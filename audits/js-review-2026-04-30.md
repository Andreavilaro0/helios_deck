## P1 — should fix (3 findings)

### `app/root.tsx:21-23` — Google Fonts es un stylesheet render-blocking

**Problem:** `<link rel="stylesheet" href="https://fonts.googleapis.com/...">` bloquea el render hasta que la hoja de estilos externa se descarga. Viola el principio de CLAUDE.md "No render-blocking scripts" (aplica igual a stylesheets). Cargar fuentes desde Google Fonts tiene implicaciones GDPR (petición a servidor externo por cada visita de usuario EU).

**Fix aplicado:** Eliminar las 3 entradas de Google Fonts de `links()`. Reemplazar en `app.css` con system font stack. Decisión sobre self-hosted fonts (`@fontsource/inter`) aplazada a Phase 5 (diseño visual).

**Why:** Performance + privacidad. `preconnect` mitiga latencia pero no elimina el bloqueo de render. Afecta LCP y la checklist de performance del evaluador.

---

### `app/routes/home.tsx:5-8` — Meta tags con contenido de scaffold

**Problem:** `title: "New React Router App"` y `description: "Welcome to React Router!"` son placeholder del scaffold. Cualquier evaluador que abra la URL verá este título en el navegador y en la pestaña.

**Fix aplicado:**
```tsx
// before
{ title: "New React Router App" },
{ name: "description", content: "Welcome to React Router!" },

// after
{ title: "HELIOS_DECK — Space Weather Observatory" },
{ name: "description", content: "A fullstack observatory for normalized heliophysical and geophysical data." },
```

**Why:** Parte de la entrega evaluable. El `<title>` es lo primero que ve cualquiera que abra DevTools o la pestaña del navegador.
